fn run_zero_membrane(&mut self)

Starts a node without requiring responses from the network. Starts the routing membrane without looking to bootstrap. It will relocate its own address with the hash of twice its name. This allows the network to later reject this zero node when the routing_table is full.

A zero_membrane will not be able to connect to an existing network, and as a special node, it will be rejected by the network later on.
